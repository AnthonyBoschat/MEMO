Moteur transactionnelle
Ce type de moteur est plus lent que le moteur classique, mais offre une sécurité lors du traitement des données
Contre les pannes par exemple, ils feront un rollback arrière si les fichier sont corrompus

exemple : InnoDB

  • Beaucoup d'écritures
  • Sécuriser les données
  • Cohérence entre les données ( tables )
Moteur classique
Ne possède pas de sécurité comme les moteur transactionnelles, mais sont plus rapide.

exemple : MyISAM

  • Rapide en écriture, très rapide en lecture ( beaucoup de lecture )
  • Efficace pour la recherche
  • Bloque la table lors de l'enregistrement
Changer de moteur
SET default_storage_engine = InnoDB;

ou

SET default_storage_engine = MyISAM;

Contrainte de clef primaire ( permet d'identifier de manière unique chaque enregistrement, garantie l'unicité et l'identification des données)
PRIMARY KEY(#colonne_unique)

Génère automatiquement une valeur croissante dans une colonne lorsque lors de l'insertion d'une nouvelle ligne dans une table ( Il faut qu'une clef primaire ou unique ai été déclarer Il ne peut y avoir qu'une seule colonne en auto_increment)
AUTO_INCREMENT

Assigne une clef etrangere
FOREIGN KEY (#nom_colonne_clef_etrangere) REFERENCES #nom_tableau(#nom_colonne_du_tableau)

Pour indiquer une colonne obligatoire :
MODIFY user_name varchar(30) NOT NULL;
Pour annuler ceci :
MODIFY user_name varchar(30);


Pour indiquer une valeur par défaut :
ALTER user_admin SET DEFAULT 0;
Pour supprimer une valeur par défaut :
ALTER user_admin DROP DEFAULT;


Pour indiquer une vérification de valeur :
ADD CHECK(user_id <= 1000 AND user_id > 0);
( Le nom de cette contrainte sera par défaut : table_users_chk_1 ) => première contrainte check renseigner dans le tableau "table_users"
Pour lui donner un nom spécifique :
ADD CONSTRAINT chk_nombre_trop_grand CHECK(user_id <= 10000 AND user_id > 0 AND user_admin < 1);
( Le nom de cette contrainte est : chk_nombre_trop_grand )
Pour supprimer une contrainte check:
DROP CHECK #nom_de_la_contrainte_check;


Pour indiquer une clef unique :
ADD UNIQUE(user_name, user_id);
( Le nom de la contrainte est par défaut le nom de la première colonne indiquer : user_name )
Pour donner un nom spécifique :
ADD CONSTRAINT unique_key UNIQUE(user_id, user_name);
Pour supprimer une contrainte unique :
DROP INDEX #nom_de_la_contrainte_unique;


Pour indiquer une clef primaire :
ADD PRIMARY KEY (user_id, user_name);
Pour supprimer les clef primaire (Il faut qu'il n'y ai plus d'auto-incrémentation )
DROP PRIMARY KEY;


Pour indiquer une auto_incrémentation (Il faut qu'une clef primaire ou unique ai été renseigner ) :
MODIFY user_id smallint(6) not null AUTO_INCREMENT;
Pour supprimer l'auto-incrémentation :
MODIFY user_id smallint(6) not null;


Pour indiquer une clef étrangere
ADD FOREIGN KEY (news_author) REFERENCES table_users(user_id)
Par défaut, le nom de la contrainte sera : table_news_ibfk_1 (ibfk = InnoDBForeignKey)
Pour lui donner un nom spécifique
ADD CONSTRAINT clef_etrangere FOREIGN KEY (news_author) REFERENCES table_users(user_id)
Le nom de la contrainte sera : clef_etrangere
Pour supprimer une clef etrangere : supprimer sa contrainte => supprimer la clef
DROP CONSTRAINT #nom_de_la_contrainte;
DROP KEY #nom_de_la_clef;
Par défaut, le nom de la clef est le nom de la colonne ou a été appliquer la clef, ici : news_author


Assigner un index NE PAS RENSEIGNER ALTER TABLE EN AMONT :
CREATE INDEX index_utilisateur ON table_users(user_id, user_name);
Supprimer un index
DROP KEY index_utilisateur ( nom de l'index );
ou bien
DROP INDEX index_utilisateur ( nom de l'index );

Documentation
Accorder un privilège

GRANT #privilège1, #privilège2
ON #endroit_à_appliquer ( *.* => tout les tableau de toute les bases de données )
TO "anthony"@"localhost";

Pour indiquer que l'utilisateur peut aussi accorder ou revoquer des privilège on termine la ligne de commande par :
WITH GRANT OPTION;
Révoquer un privilège

REVOKE #privilège1, #privilège2
ON #endroit_à_appliquer ( *.* => tout les tableau de toute les bases de données )
FROM "anthony"@"localhost";

Pour signifier de retirer tout les privilège d'un utilisateur on peut indiquer au début :
REVOKE ALL

Pour supprimer le droit de grant option
REVOKE GRANT OPTION
ON #endroit_à_appliquer
FROM "anthony"@"localhost";
Exemple

GRANT CREATE
ON cours_sql.* ( toute les tables de la base de donnée cours_sql )
TO "anthony"@"localhost"
WITH GRANT OPTION;


Et pour supprimer tout ceci
REVOKE GRANT OPTION, CREATE
ON cours_sql.*
FROM "anthony"@"localhost";


Dans tout les cas, même tout les privilèges supprimer, il gardera au moin un droit d'usage

Indiquer une plage :

Avec IN

Imaginons que nous voulions tout les enregistrement des jeux qui ont un stock de 2, 4, et 6.
Pour eviter un code à rallonge comme :
SELECT * FROM fv_games WHERE game_stock = 2 OR game_stock = 4 OR game_stock = 6;

On peut raccourcir le code :
SELECT * FROM fv_games WHERE game_stock IN(2, 4, 6);

Affiche toute les informations des enregistrement dans la table fv_game lorsque le stock du jeu est 2, 4, ou 6.

Avec BETWEEN

Imaginons que nous voulions tout les enregistrement des jeux quit ont un ID compris entre 5 et 15 ( inclu )
Pour eviter un code à rallonge comme :
SELECT * FROM fv_games WHERE id_game >= 5 AND id_game <= 15 ;

On peut raccourcir le code :
SELECT * FROM fv_games WHERE id_game BETWEEN 5 AND 15;
( Attention, BETWEEN inclus les nombres indiquer )
Rechercher par vérifications de chaînes :

SELECT * FROM fv_games WHERE game_title LIKE "pok%";

Affiche toute les informations des enregistrement dans la table fv_game lorsque le titre du jeu commence par les lettres : pok
( Le symbole % indique qu'il peut s'agir de n'importe quoi ensuite, n'importe combien de caractère )
( Le symbole _ indique qu'il peut s'agir de n'importe quoi ensuite, du moment que ceci colle directement à la suite la chaîne renseigner pok )
N'est pas null ou est null

SELECT * FROM fv_games WHERE game_price IS NOT NULL;

ou

SELECT * FROM fv_games WHERE game_price IS NULL;

Affiche toute les informations des enregistrement dans la table fv_game lorsque le prix du jeu est null
Triée par ordre alphabétique

SELECT * FROM fv_games ORDER BY game_title ADC/DESC;

Affiche toute les informations des enregistrement dans la table fv_game en les affichant dans l'ordre de leurs titres ( Alphabétique )
Par défaut, le triage s'effectue sur l'ID
ASC/DESC = Ascendant ou descendant ( par défaut Ascendant )
Limité le nombre de lecture

SELECT * FROM fv_games LIMIT 5;

Affiche toute les informations des 5 premiers enregistrement dans la table fv_game

Attention, LIMIT est différent selon certains SGBD ( Systèmes de gestion de base de données )
MySQL, PostgreSQL, MariaDB : LIMIT
SQL Server, MS Access : TOP
Oracle : ROWNUM

GROUP BY
Par exemple, afficher le nombre d'utilisateur pour chaques pays :
SELECT COUNT(*) as "Nombre de résident", user_country as "Pays" FROM fv_users GROUP BY user_country;

Le COUNT et le nom des pays est GROUPER par le nom des pays


Par exemple, afficher la somme totale des âges des résident par pays :
SELECT SUM(user_age) as "Âges des résidents", user_country as "Pays" FROM fv_users GROUP BY user_country WITH ROLLUP;

WITH ROLLUP Affiche à la fin le total des âges de tout les résident, tout pays confondu


Filtrer des résultats issu d'un groupement de donnée

Par exemple, indiquer les pays ou la somme de leurs âges dépasse 20 :
SELECT user_country as "Pays", SUM(user_age) as "Âges des résidents" FROM fv_users GROUP BY user_country HAVING SUM(user_age) > 20;

UNION
Concaténer le résultat de plusieurs requêtes

Au lieu d'avoir deux requêtes :

SELECT * FROM fv_site1_users;
SELECT * FROM fv_site2_users;
on peut faire :

SELECT * FROM fv_site1_users
UNION
SELECT * FROM fv_site2_users;

• Il faut que le nombre de colonnes à récupérer soit égal dans les deux requêtes
• Il faut que l'ordre donner aux colonnes soit le même
• Il faut que le type de donnée soit la même

• Ne récupère pas les doublons identiquement parfait

( Si on veut récupérer tout, même les doublons parfait )

SELECT * FROM fv_site1_users
UNION ALL

Exemple
Imaginons que je veuille connaitre toute les abilités des joueurs inscrit sur le jeu, je pourrais faire :

select * from fv_players;

Cependant, nous ne verrions pas leurs abilities ( qui est un numéro ), c'est à ça que servira par exemple une jointure :

SELECT player_name, player_level, ability_name
FROM fv_players
INNER JOIN fv_abilities ON player_ability = id_ability;
  • Affiche moi player_name, player_level, ability_name
  • Depuis la table fv_players
  • ability_name n'existe pas dans la table fv_players, donc on relie cette table avec la table qui a cette colonne ( INNER JOIN fv_abilities )
  • Affiche moi donc maintenant le ability_name correspondant quand player_ability ( un numero ) = id_ability ( un numero )


Imaginons maintenant que je veuille afficher tout les joueurs qui n'ont jamais acheter d'objet :

SELECT player_name
FROM fv_playertrades 
RIGHT JOIN fv_players
ON playertrade_buyer = id_player
WHERE playertrade_buyer IS NULL;
  • Tu va m'afficher des noms (SELECT player_name)
  • En te servant de la table des transactions (FROM fv_playertrades)
  • Les noms sont ici (RIGHT JOIN fv_players)
  • Tu va m'afficher des noms en comparant deux id (ON playertrade_buyer = id_player)
  • Et enfin, je veux que tu m'affiche le nom du joueur ( player_name ) Qui n'a pas d'ID d'acheteur (WHERE playertrade_buyer IS NULL)
Jointure multiple
Je veux afficher le nom des joueurs vendeur, les objets vendu, et les joueur acheteur

SELECT
vendeur.player_name AS "joueur vendeur"
objet.item_name AS "objet vendu"
acheteur.player_name AS "joueur acheteur"

FROM fv_playertrades

INNER JOIN fv_players AS vendeur
ON playertrade_seller = vendeur.id_player

INNER JOIN fv_items AS objet
ON playertrade_item = objet.id_item

INNER JOIN fv_players AS acheteur
ON playertrade_buyer = acheteur.id_player;

Dans ce cas de figure, nous avons à lié deux fois la table fv_players, pour deux utilisations différentes, on utilise donc des alias pour les distinguer
Jointure profonde
Je veux afficher le nom des classes des joueurs vendeur

SELECT ability_name
FROM fv_playertrades

INNER JOIN fv_players
ON id_player = playertrade_seller

INNER JOIN fv_abilities
ON id_player = id_ability;
  • En te servant de la table des transactions (FROM fv_playertrades)
  • Tu vas m'afficher des noms de classes (SELECT ability_name)
  • Join moi la table des joueurs (INNER JOIN fv_players)
  • Et relie moi les ID des vendeurs avec les ID des joueurs (ON id_player = playertrade_seller)
  • Join moi la table des classes (INNER JOIN fv_abilities)
  • Et relie moi les ID des joueurs avec les ID des classes (ON id_player = id_ability)

• Peut revenir en arrière avec la clé

Hashage
Donnée en clair -> Donnée hasher ( un hash )

• Ne peut pas revenir en arrière
Hashage

-> 32 caractères (128 bits)
NE PAS UTILISER

SELECT SHA("bonjour");
NE PAS UTILISER




SELECT SHA2("bonjour", #bits);
Conseiller
#bits :
  • 224 (56 caractères)
  • 256 (64 caractères)

  • 512 (128 caractères)
